# measurement_app A版 MVP 功能規格

## 專案概述

| 項目 | 內容 |
|------|------|
| **產品概念** | 將任意物體放在信用卡旁邊 → 拍張照 → AI 立即算出物體長度與寬度 |
| **MVP 形式** | Web 版（手機瀏覽器即可使用，不需安裝 App） |
| **驗證目標** | 驗證「信用卡參考物 + AI 物體偵測」的量測精準度 |
| **建立日期** | 2026-05-14 |

---

## 核心原理

1. **信用卡作為比例尺**：利用信用卡的標準尺寸（**8.5cm**）作為 pixel-to-cm 的換算基準
2. **AI 偵測任意物體**：AI 自動辨識照片中的任意物體輪廓（不限定魚，任何東西都可以）
3. **AI 計算長度與寬度**：根據物體輪廓自動計算最長邊（長度）與最寬邊（寬度），再透過信用卡比例換算實際尺寸

---

## 使用流程

1. 將要測量的物體放置於平坦地面
2. **將信用卡放在物體旁邊（同一平面）**
3. 用瀏覽器開啟網頁，拍照或從相簿選取
4. **AI 自動偵測**：
   - 信用卡邊界（計算 pixel-to-cm 比例）
   - 任意物體輪廓（偵測物體範圍）
   - 物體長軸與短軸（計算長度與寬度）
5. 透過信用卡比例換算實際 cm
6. 顯示結果並記錄環境資訊

## 技術架構

### 前端
- **框架**：React / Vue
- **相機存取**：MediaDevices API
- **圖片處理**：Canvas API
- **GPS**：Geolocation API
- **EXIF 讀取**：exif-js
- **資料儲存**：IndexedDB / LocalStorage

### 後端 / AI
- 圖片上傳與處理 API
- **AI 信用卡偵測**：YOLO / OpenCV 邊緣偵測 / 模板匹配
- **AI 物體偵測**：SAM (Segment Anything Model) — 偵測任意物體輪廓
- **AI 長寬軸計算**：在物體輪廓上自動計算最長邊與最寬邊
- **比例換算**：信用卡 pixel 長度 ÷ 8.5cm = 比例因子 → 物體 pixel 長寬 × 比例因子 = 實際長寬
- 天氣 API 整合（OpenWeatherMap）

## 預期精準度

| 條件 | 預期誤差 |
|------|----------|
| 理想條件（正上方、同一平面、光線充足、信用卡與物體不重疊）| ±2-5% |
| 一般條件（些許角度偏差）| ±5-10% |
| 較差條件（大角度、物體彎曲）| ±10-20% |

---

## 限制與假設

- 信用卡與被測量物體必須在同一平面上
- 建議相機盡量垂直於地面拍攝（減少透視誤差）
- 信用卡預設尺寸為 **8.5cm**（可設定調整）
- AI 偵測的準確度取決於物體與背景的對比度
- 物體彎曲時，AI 會回傳最長直線距離（非弧長）
