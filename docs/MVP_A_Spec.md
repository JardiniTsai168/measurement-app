# measurement_app A版 MVP 功能規格

## 專案概述

| 項目 | 內容 |
|------|------|
| **產品概念** | 釣到魚 → 放地上 → 放信用卡 → 拍張照 → 立即得知魚長度 + 環境記錄 |
| **MVP 形式** | Web 版（手機瀏覽器即可使用，不需安裝 App） |
| **驗證目標** | 驗證「信用卡參考物」方案的量測精準度 |
| **建立日期** | 2026-05-14 |

---

## 核心原理

利用信用卡的標準尺寸（**85.6mm × 54.0mm**）作為比例參考，透過影像分析計算魚的實際長度。

---

## 使用流程

1. 釣到魚，放置於平坦地面
2. **將信用卡放在魚旁邊（同一平面）**
3. 用瀏覽器開啟網頁，拍照或從相簿選取
4. AI 自動偵測信用卡邊界
5. AI 自動偵測魚頭到尾鰭的長度（pixel 距離）
6. 透過信用卡 pixel 尺寸換算實際 cm
7. 顯示結果並記錄環境資訊

## 功能規格

### 1. 照片上傳
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 相機拍攝 | MediaDevices API 呼叫手機相機即時拍攝 | P0 |
| 相簿選取 | 從相簿選擇現有照片 | P0 |
| 照片預覽 | 拍攝後顯示預覽，確認或重拍 | P0 |

### 2. 信用卡偵測
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 自動偵測信用卡邊界 | 辨識照片中的信用卡矩形邊界 | P0 |
| 信用卡尺寸校正 | 以 85.6mm 為基準計算 pixel-to-cm 比例 | P0 |
| 手動調整邊界 | 用戶可微調自動偵測的信用卡框 | P2 |

### 3. 魚體量測
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 魚體偵測 | 自動框選魚輪廓 | P0 |
| 魚頭定位 | 標記魚頭前端位置 | P0 |
| 尾鰭定位 | 標記尾鰭末端位置 | P0 |
| 長度計算 | 計算頭尾直線距離，換算出實際 cm | P0 |
| 手動調整頭尾 | 用戶可微調自動標記的頭尾點 | P1 |
| 曲線處理 | 魚身彎曲時計算弧長（不只直線） | P2 |

### 4. 環境記錄（自動抓取）
| 功能 | 資料來源 | 說明 | 優先級 |
|------|----------|------|--------|
| GPS 位置 | 照片 EXIF / Geolocation API | 拍攝地點座標 | P0 |
| 拍照時間 | 照片 EXIF / 裝置時間 | 自動記錄 | P0 |
| 氣溫 | OpenWeatherMap API | 用 GPS 座標查詢 | P1 |
| 濕度 | OpenWeatherMap API | 同上 | P1 |
| 風向 | OpenWeatherMap API | 同上 | P1 |

### 5. 結果呈現
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 長度顯示 | 顯示魚的總長度（cm） | P0 |
| 量測標示 | 在照片上畫出魚頭到尾鰭的標示線 | P1 |
| 資訊卡片 | 整合照片 + 長度 + GPS + 時間 + 天氣的分享卡 | P1 |
| 儲存圖片 | 下載資訊卡片到裝置相簿 | P1 |

### 6. 拍照引導
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 信用卡擺放提示 | 提示用戶將信用卡與魚放同一平面 | P1 |
| 角度偏差提示 | 偵測拍攝角度過大並提示重拍 | P2 |
| 光線檢測 | 偵測畫面過暗或過曝 | P2 |

### 7. 歷史記錄
| 功能 | 說明 | 優先級 |
|------|------|--------|
| 量測列表 | 顯示過往量測記錄（IndexedDB） | P2 |
| 記錄詳情 | 查看單筆完整資訊 | P2 |

## 技術架構

### 前端
- **框架**：React / Vue
- **相機存取**：MediaDevices API
- **圖片處理**：Canvas API
- **GPS**：Geolocation API
- **EXIF 讀取**：exif-js
- **資料儲存**：IndexedDB / LocalStorage

### 後端 / AI
- 圖片上傳與處理 API
- 信用卡偵測模型（YOLO / OpenCV）
- 魚體偵測模型（SAM / YOLO-seg）
- 長度計算演算法
- 天氣 API 整合（OpenWeatherMap）

---

## 預期精準度

| 條件 | 預期誤差 |
|------|----------|
| 理想條件（正上方、同一平面、光線充足）| ±2-5% |
| 一般條件（些許角度偏差）| ±5-10% |
| 較差條件（大角度、彎曲魚身）| ±10-20% |

---

## 限制與假設

- 信用卡與魚必須在同一平面上
- 建議相機盡量垂直於地面拍攝（減少透視誤差）
- 魚身彎曲時，直線長度會低估實際長度
- 信用卡全球統一尺寸 85.6mm（如支援其他國家需確認尺寸）
